# CAMA 홈페이지 수정본 업로드 가이드

이 파일은 이미 만든 GitHub 저장소와 Vercel 홈페이지에 수정본을 반영하는 절차입니다.

## 현재 적용할 수정사항

1. 도메인은 구매하지 않고 `https://cama-website.vercel.app/` 주소를 그대로 사용합니다.
2. 메인 표지에 `주관·주최 | 아하바프레이즈선교회`를 표시합니다.
3. `CAMA는 무엇인가요?` 문구에 `아하바프레이즈선교회(C&MA 얼라이언스 한국총회 소속)가 행정 주최 단체로 함께합니다.`를 반영합니다.
4. 아하바프레이즈선교회는 협력단체가 아니라 **행정 주최 단체**로 표시합니다.
5. 협력단체에 **CAMA Station(문화예술선교연합아카데미)**, **MJ댄스컴퍼니(명지대미래교육원 선교무용단)**를 추가합니다.
6. 협력단체의 `토브`를 **토브 앙상블**로 수정합니다.
7. 연락처/문의 섹션은 삭제합니다.
8. 행사 아카이브 섹션을 추가하고, 아래 행사 배너를 넣습니다.
   - HAPPY NEW YEAR TANZANIA
   - 5월 가정의 달 사랑나눔콘서트
   - 꿈꾸는 글로벌 평화통일
   - Bible Concept Exhibition
9. `HAPPY NEW YEAR TANZANIA`를 누르면 상세 페이지로 이동하도록 구성합니다.

## GitHub에 올리는 방법

1. 이 ZIP 파일을 다운로드합니다.
2. 압축을 풉니다.
3. GitHub 저장소로 들어갑니다.
   - `https://github.com/davidjoon101349-create/CAMA-WEBSITE`
4. `Add file` → `Upload files`를 누릅니다.
5. 압축을 푼 폴더 안의 파일들을 업로드합니다.
   - `index.html`
   - `happy-new-year-tanzania.html`
   - `style.css`
   - `script.js`
   - `assets` 폴더 전체
   - `README.md`
   - `robots.txt`
   - `sitemap.xml`
   - `초보자_배포_가이드.md`
6. 아래 커밋 메시지를 입력합니다.

```text
Add event archive and Tanzania detail page
```

7. `Commit changes`를 누릅니다.

## Vercel 반영 확인

GitHub에 커밋하면 Vercel이 자동으로 다시 배포합니다. 보통 배포가 끝나면 아래 주소에서 수정본을 확인할 수 있습니다.

```text
https://cama-website.vercel.app/
```

상세 페이지 주소는 아래와 같습니다.

```text
https://cama-website.vercel.app/happy-new-year-tanzania.html
```

## 확인할 부분

- 메인 표지에 `주관·주최 | 아하바프레이즈선교회`가 표시되는지 확인합니다.
- `CAMA는 무엇인가요?` 문구에 행정 주최 단체 설명이 들어갔는지 확인합니다.
- 상단 메뉴에 `행사 아카이브`가 있고, 하위 메뉴에 `행사 소개`, `HAPPY NEW YEAR TANZANIA`가 보이는지 확인합니다.
- 행사 아카이브 섹션에 4개 행사 배너가 보이는지 확인합니다.
- `HAPPY NEW YEAR TANZANIA` 배너를 눌렀을 때 상세 페이지로 이동하는지 확인합니다.
- 협력단체 섹션에서 아하바프레이즈선교회가 `행정 주최 단체`로 따로 표시되는지 확인합니다.
- 협력단체에 `CAMA Station`, `MJ댄스컴퍼니`, `토브 앙상블`이 표시되는지 확인합니다.
- 연락처 이메일이나 임시 문구가 남아 있지 않은지 확인합니다.
